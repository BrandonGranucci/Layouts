You could change each container's rank or size by setting their flex to varying numbers (ex: flex:1). The closer to one, the smaller the element
compared to other elements. If each element has the same flex number, there is no variation between them.

You could change the order of an element in its container with order: (number). This allows you to not need to restructure your HTML elements.
The boxes in my HTML code are in the order of box 1, then 2, then 3 but by using the order property
I can have them in a different order.

You could also align container items at the top with align-items:flex-start or at the bottom with align-items:flex-end.

If you want there to be space/margins between the items in your flex box, you need to use justify-content:space-between on your entire container. Also,
this would mean your boxes would have to be certain sizes, which can be set with width:(number)%.

Considering that these columns are too large to be displayed on smaller screens like with phones, you may want to only allow these columns to occur with the @media property.
If you set a certain min-width:(# of pixels)px inside the @media function, and wrap it around the column in the css file,
the columns won't be created. You could see the columns vanish if you shrink the browser.


If you want to ensure that your elements wrap around to the next line, use the flex-wrap:wrap property inside of your container.
